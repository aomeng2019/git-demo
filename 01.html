将本地仓库推送到远程仓库命令 1.git push 远程仓库地址 分支名称 2.git push 远程仓库地址别名 分支名称 3.git push -u 远程仓库地址别名 分支名称 -u 记住推送地址及分支 下次推送只需要直接输入git push即可 为远程仓库添加别名 4.git remote add 远程仓库地址别名 远程仓库地址 remote是远程的意思 远程仓库地址别名一般为origin 第一次推送到远程仓库时要登陆，登陆后的信息就保存在你自己电脑的windows系统中的凭据管理中。下一次要将本地仓库推送到远程仓库时，windows系统自动将登陆信息发送给github.com并登陆。则就不需要用户再一次登陆了 如果远程仓库中的版本高于本地仓库中的版本，此时本地仓库是不能向远程仓库中提交的。本地仓库必须先拉取远程仓库中的版本，然后再向远程仓库提交 github中除了支持团队合作之外还支持非团队合作，也就是说即使不是该团队的成员也可以向该远程仓库中提交代码。只不过不是该团队成员提交的代码是需要被审核的， 只有审核通过了才能够生效。 应用场景： 你要完成一个网页效果，但是这个效果写出来要花费一些时间。你不想花费这个时间，于是你在github中找到了一个正好能够实现这个网页效果的库（别人上传到github中的远程仓库），但是你在使用这个库的时候发现了一些不好的地方，于是你对这些对方加入了自己的修改。然后你就可以将你自己完善的代码提交给原仓库的作者，如果原仓库的作者认为你提交的代码有价值就同意将你写的代码合并到原来的仓库中。如何才能向其他的仓库中贡献代码呢？？？？？？？ 程序员C访问程序员A的仓库git-demo,并点击页面中的fork，将程序员A创建的git-demo仓库复制一份到自己的github账号中.这样程序员C的github账户中也有一个git-demo仓库。则程序员C的github账户中的git-demo仓库的所有权就属于程序员C，他可以对这个仓库做任何操作。接下来程序员C需要克隆这个仓库到本地，在本地对这个仓库进行修改。修改完成以后再推送到程序员C的github账号的git-demo仓库中。然后再向原作者程序员A发送代码修改的请求。 如何向原作者程序员A发送代码修改的请求？？？？ 在程序员C的github页面中的git-demo仓库页面点击pull requests,再点击new pull request(创建一个新的pull请求)。再点击Create pull request填写修改的功能以及具体的修改的功能的描述。再点击create pull request则系统会将程序员C修改的文件和修改的功能的描述一起发送给程序员A。 则在原作者程序员A的github账号的git-demo仓库页面中的pull requests中显示有消息。 commits表示程序员C的修改文件， files changed按钮表示文件有什么变化 跨团队协作 1.程序员C fork仓库 2.程序员C 将仓库克隆在本地进行修改 3.程序员C将仓库推送到远程 4.程序员C发起pull request 5.原仓库作者审核 6.原仓库作者合并代码 ssh登陆 生成秘钥：ssh-keygen 秘钥储存目录：c:\Users\ME\.ssh\id_rsa 公用秘钥名称：id_rsa.pub 保存在github网站中 私钥名称：id_rsa 保存在开发者的电脑中 rsa是一种非对称加密方式 git忽略清单 将不需要被git管理的文件名字添加到此文件中，在执行git命令的时候，git就会忽略这些文件。git忽略清单文件名称: .git 将工作目录中的文件全部添加到暂存区： git add . 为github中的远程仓库添加详细说明
